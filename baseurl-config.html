GitHub Pages 基础URL配置指南

此文件提供了在GitHub Pages上正确配置资源文件路径的方法，特别是当您的网站部署在子目录中时。

问题描述

当您的网站部署在GitHub Pages的子目录中（例如 username.github.io/repo-name/）时，资源文件（CSS、JavaScript、图片等）的引用路径可能会出现问题，导致404错误。

解决方案

方法1：使用相对路径（推荐）

在HTML文件中使用相对路径引用资源文件，不要以斜杠开头：

<link rel="stylesheet" href="css/style.css">
<script src="js/script.js"></script>

方法2：使用仓库名称作为基础路径

如果您知道仓库名称，可以在路径前添加仓库名称：

<link rel="stylesheet" href="/repo-name/css/style.css">
<script src="/repo-name/js/script.js"></script>

将 repo-name 替换为您的实际仓库名称。

方法3：使用base标签

在HTML的head部分添加base标签，指定基础URL：

<base href="https://username.github.io/repo-name/">

将 username 和 repo-name 替换为您的GitHub用户名和仓库名称。

注意： 使用base标签后，所有相对路径都将基于此URL，包括链接。如果您的网站有复杂的导航结构，这可能会导致问题。

检查清单

  1. 确保 .nojekyll 文件存在于仓库根目录
  2. 确保所有资源文件（CSS、JS、图片等）都已上传到正确的目录
  3. 使用浏览器开发者工具检查具体哪些资源加载失败，查看网络请求的URL是否正确
  4. 尝试清除浏览器缓存后重新访问网站
  5. 等待几分钟，GitHub Pages可能需要一些时间来部署更改

测试您的配置

修改配置后，您可以通过以下方式测试资源文件是否能正确加载：

  1. 打开浏览器开发者工具（F12或右键检查）
  2. 切换到"网络"或"Network"选项卡
  3. 刷新页面
  4. 查看CSS和JavaScript文件是否成功加载（绿色状态码200）
